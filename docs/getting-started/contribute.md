# OhMyUniversity! - How to Contribute

Contributions are welcome and follow a simple workflow.

### Basic steps

1. Fork the repository
2. Create a feature branch
3. Make changes following project conventions
4. Submit a pull request

### Guidelines

- Follow existing naming conventions
- Keep changes focused and minimal
- Document any new API or data source additions
- Ensure consistency across repositories

### Commit style

Use clear and descriptive commit messages.

Example:

```
add MIUR dataset mapping for enrollments
```

### Where to start

Look for small improvements in:

- documentation
- dataset descriptions
- API consistency
- missing metadata
